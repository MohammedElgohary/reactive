computedAsync()

Async derived values with automatic re-runs, loading/error states, and built-in race-condition protection.

User lookup
Pick a user ID — the async computed re-runs automatically and tracks loading / error state.

Race condition protection
Rapid clicks fire multiple requests. Only the last one updates the UI — stale responses are silently discarded.

Error handling
Enter an ID > 10 to trigger a 404 — the error state is exposed directly on the async computed.
How it works
const state = reactive({ userId: 1 });

const user = computedAsync(async () => {
  const id = state.userId;          // sync read -- tracked as dependency
  const res = await fetch('/api/users/' + id);
  if (!res.ok) throw new Error('HTTP ' + res.status);
  return res.json();
}, { initialValue: null });

// Reactive state exposed on the result:
user.value    // resolved data (or initialValue)
user.loading  // true while in-flight
user.error    // Error | null

// Re-runs automatically when state.userId changes.
// If userId changes again before the previous fetch
// completes, the stale response is discarded.
