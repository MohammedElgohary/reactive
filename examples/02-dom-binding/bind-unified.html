Unified bind()

One function for every binding type — auto-detects text vs HTML, supports prefix syntax for class, style, and attributes.

Why bind()?
Instead of importing and calling a different function for each binding type, bind() handles everything through a consistent interface.
// Specific functions
bindText('#title', () => state.title);
bindAttr('#link', 'href', () => state.url);
bindClass('#btn', 'active', () => state.on);
bindStyle('#box', 'color', () => state.color);

// Unified — same result
bind('#title', () => state.title);
bind('#link', 'href', () => state.url);
bind('#btn', 'class:active', () => state.on);
bind('#box', 'style:color', () => state.color);
1. Auto-detection — text vs HTML
Without a second argument, bind() sets textContent. Pass { trusted: true } to render HTML from developer-controlled templates.
2. Attribute binding
Pass the attribute name as the second argument — URL validation and dangerous-attribute blocking apply automatically.
Demo Link
3. Class & style prefix syntax
Use class:name for boolean class toggles and style:property for inline styles.
Box
4. Security — trusted vs untrusted
User input is always sanitized. Use { trusted: true } only for templates you control — never for user-provided content.
