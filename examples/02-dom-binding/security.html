Security — XSS Protection

Built-in protection against Cross-Site Scripting. Every DOM binding sanitizes by default — opt in to trust, never opt out of safety.

1. Default sanitization
bindHTML() and render() sanitize HTML automatically. Scripts, event handlers, and dangerous tags are stripped.

2. escapeHtmlEntities() vs sanitizeHtmlContent()
Escape converts all HTML to entities — shows as literal text. Sanitize strips dangerous parts but keeps safe markup.

3. { trusted: true } — when and why
Use trusted: true only for templates you wrote. Never for user input, URL params, or API data you don't control.
✅ Safe to trust:
  • Templates in your source code
  • Static HTML from your build
  • CMS content with server-side sanitization
❌ Never trust:
  • User comments or form input
  • URL query parameters
  • Data from third-party APIs
4. User data inside trusted templates
When embedding user values inside a trusted template, escape them with escapeHtmlEntities() first.
5. isUrlSafe() — block javascript: URLs
bindAttr() validates URLs for href, src, and action automatically. Use isUrlSafe() for manual checks.
URL Safe?
https://example.com ✅ Yes
/relative/path ✅ Yes
javascript:alert('xss') ❌ No
data:text/html,<script>… ❌ No
6. Blocked attributes & properties
bindAttr() blocks all on* event handlers. bindProp() blocks innerHTML and similar.
